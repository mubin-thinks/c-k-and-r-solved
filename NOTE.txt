1. Varation of main function
----------------------------
In the K&R book, the version of C used is known as "K&R C".
K&R C uses implicit int as return type for functions.
Therefore, the main function looks like `main()` instead
of `int main()`.

In later versions of C, `int main()` is the norm. And you
should also explicitly declare return type of functions.
Today's compilers have removed/discourage the use of
`main()`.

In the solutions, the return type of all functions are
declared & main always returns 0(if execution is
successful).



2. Testing various input to `getchar()`
---------------------------------------
For exercises using getchar(), providing EOF as input to the
terminal stdin input stream is difficult. instead there's a
much better way using files.

Creating a file(like text files) and using the `cat` program
will ouput the contents of the file. This causes the EOF
symbol to also be printed.

In UNIX we can use the |(pipe) symbol to forward the output
of cat as input to our program like so(at least in Linux):

        cat input.txt | ./a.out
